# Dolores Design System

A Storybook-driven workbench for **Untitled UI React** components, styled with **Tailwind CSS v4** and built with **TypeScript** on top of **Vite + React 19**.

## What's wired up

- **Theme switcher** in the toolbar — toggles `class="dark-mode"` on `<html>` so every story renders in light or dark.
- **Accessibility tab** — every render is checked by `axe-core` via `@storybook/addon-a11y`.
- **Interactions** — `play()` functions in stories use `userEvent`, `expect`, and `fn()` from `storybook/test`. Stepped through in the Interactions panel.
- **Docs page** — every component has an autogenerated docs page with a props table built from TypeScript types and live, editable Controls.
- **MCP server** — `@storybook/addon-mcp` exposes an HTTP MCP endpoint while `storybook dev` is running. AI agents can introspect stories and run them.

## Where things live

- `src/components/base/**` — Untitled UI components (you own the source).
- `src/components/base/**/*.stories.tsx` — story files, co-located with their components.
- `src/styles/globals.css` — Tailwind v4 entry + Untitled UI theme tokens.
- `.storybook/main.ts` — addon registration.
- `.storybook/preview.ts` — global decorators, theme switcher, controls config.

## Adding a new component

```bash
npx untitledui@latest add <component-name>
```

Then create `<component>.stories.tsx` next to it following the patterns in `Base/Buttons/Button` or `Base/Input/Input`.
